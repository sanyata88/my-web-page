Chapter 1:World Wide Web and HTML

Chapter 1.1. Workings of the web

The computer or browser communicates with various servers through the web by using HTTP.HTML is the main type of document of the web and stands for hyper text markup language .HTML constitutes text, links and references such as videos and images.Markup defines what the content looks like.

Chapter 1.2.Tags and Elements

Tags may be nested within tags.Most tags need to be opened and then closed. Not closing a tab will apply the function to the entire text.Editors such as scratchpad.io enable automatic closing of tags.Void tags such as break (br) do not require to be closed. Following is a short list Of Tags learnt so far:

  1. Line breaker tag for moving to next line denoted by <br>
  2. Bold tag for making text bold or highlighted is denoted by <b>
  3. Emphasize or italics by <em>
  4. To incorporate a link to a website, <a> is used.This is the link to visit the google website
  5. To add images, use the <img>.Example:This is an image of a dog:

  6. Paragraph tag boxes content within an invisible boundary.It is denoted by <p>.

Chapter 1.3. Communicating with Computers

Computers are often called "stupid" by programmers.They do not understand nuances of the human language and are less forgiving.Therefore, programmers have to be careful to use language understood by the machine.

Chapter 1.4. Inline vs Block

Tags can be classifed into these two categories. Block tags enable the developer to encompass contents inside a box within a definite, yet invisible boundary. Examples include div,p, form etc.
Inline tags do not define any such boundaries. They can help the developer move down to the next line. Examples are <br>,span, anchor, strong etc.

Chapter 2:Structure and Style

Chapter 2.1. Difference between HTML and CSS

CSS is cascading style sheets. Both CSS and HTML are languages. While HTML defines "structure" of a webpage,CSS defines the "style". Analogy could be to the components of a house.In order to build a house, we will need both these languages.

HTML will determine the basic framework or model of the exteriors and rooms of the house. CSS will be the interior designer responsible for the color of the paint, carpet, furniture etc.

You may choose to incorporate css within the html code. However, for more complex designs, it is advisable to create a separate css document within the same directory and reference the css at the beginning of the html code.

Chapter 2.2. Document Object Model:DOM

HTML follows a tree like structure for organizing content on the page. Each element is assigned a box and one can add boxes within boxes. The use of the div tag is helpful to do so. Developers often boxify the web design prior to coding.Click here for more info on DOM!

Chapter 2.3. Redundancy

Programmers must avoid repetitions or code redundancy at all costs.This will reduce the number of mistakes and improve consistency on the webpage. Repeating the same code again and again also decreases efficiency and increases amount of time to write the code. Defining a class and specifying attributes for that class can prevent us from having to write the same code again and again.
